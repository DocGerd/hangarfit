# §9 Architecture Decisions

This project records its consequential architectural decisions as
[Architecture Decision Records](https://adr.github.io/madr/) in
[**`docs/adr/`**](../adr/).

See [`docs/adr/README.md`](../adr/README.md) for the index, the
authoring workflow, and the [meta-ADR](../adr/0000-record-architecture-decisions.md)
explaining why we keep ADRs at all.

ADRs are first-class records: they live alongside the code they
describe, name the alternatives that were rejected, and stay readable
when the codebase moves on. If you are about to make a non-trivial
architectural change to `hangarfit`, the right deliverable is an ADR,
not a comment in the PR description.
